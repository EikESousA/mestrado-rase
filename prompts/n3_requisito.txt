Extrator RASE N3 (requisito).
Use APENAS o operador N2 de requisito para gerar properties_n3.
Texto completo e Texto N1 sao apenas referencia de desambiguacao.

Formato obrigatorio de saida:
- Retorne SOMENTE um JSON valido com as chaves:
  type, object, property, comparation, target, unit
- Nao retorne markdown nem texto extra.

Regras:
1) Se o operador N2 estiver vazio, retorne todos os campos vazios.
2) Se o operador N2 NAO estiver vazio, type deve ser "requisito".
3) Mapeamentos prioritarios:
   - "acessivel/acessiveis" -> property "acessível", comparation "=", target "VERDADEIRO"
   - "conectado/vinculado ... rota acessivel" ->
     property "conectado a rota acessível", comparation "=", target "VERDADEIRO"
   - "livre de obstaculos" -> property "possui obstáculo", comparation "=", target "FALSO"
   - "servida por rota acessivel" -> property "uso", comparation "=", target "VERDADEIRO"
4) Se nao houver propriedade clara, mantenha type e deixe os demais vazios.
5) unit so deve ser preenchido quando houver unidade explicita.

Exemplo 1:
Operador N2 (requisito):
"devem permanecer livres de quaisquer obstaculos de forma permanente"
Resposta:
{{
  "type": "requisito",
  "object": "porta; passagem",
  "property": "possui obstáculo",
  "comparation": "=",
  "target": "FALSO",
  "unit": ""
}}
